<h2>🎯 Architectural Patterns I Reach For</h2>

- 🔐 **Row Level Security (RLS) by default** — every tenant-isolated table enforces auth.uid() policies
- 🧾 **Append-only audit logs** — every state machine (approvals, conversions, KPI changes) ships with a paired `*_logs` table
- 🌐 **i18n via React Context** — no hardcoded UI strings; bilingual ID/EN as a first-class concern
- 🧪 **Mock = contract** — frontend types double as DB schema specs; mapping layer translates DB row ↔ FE type
- ⚡ **Realtime where it matters** — Supabase Realtime channels for notifications, presence, live status
- 📊 **Decision-grade dashboards** — Recharts + Leaflet, every chart traces back to a transparent calculation
- 🚦 **Verification workflows over hard deletes** — pending → verified / rejected / needs_revision, never destructive
- 📨 **Email failures are non-blocking** — transactional emails (Resend) log on failure but never break user flow
